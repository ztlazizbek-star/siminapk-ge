# Инструкция по загрузке обновленного sms.php на сервер

## Важно! 

Файл `public/sms.php` в этом проекте НЕ используется Next.js автоматически. Вам нужно вручную загрузить его на ваш сервер tajstore.ru.

## Шаги для загрузки:

### 1. Скачайте файл sms.php
Найдите файл `public/sms.php` в этом проекте.

### 2. Подключитесь к серверу
Используйте FTP клиент (FileZilla, WinSCP) или SSH для подключения к серверу tajstore.ru.

### 3. Загрузите файл
Загрузите файл в директорию:
\`\`\`
/path/to/tajstore.ru/simin/sms.php
\`\`\`

### 4. Проверьте настройки
Откройте загруженный файл на сервере и убедитесь, что:

\`\`\`php
$DEV_MODE = false; // Должно быть false для продакшена
\`\`\`

### 5. Проверьте права доступа
Убедитесь, что файл имеет правильные права доступа (обычно 644):
\`\`\`bash
chmod 644 sms.php
\`\`\`

## Что изменилось в новом файле:

1. **Поддержка двух типов сообщений:**
   - `type: "verification"` - Для кодов подтверждения (добавляет префикс "Ваш код подтверждения:")
   - `type: "notification"` - Для уведомлений о заказе (отправляет сообщение без префикса)

2. **Отключен DEV_MODE:**
   - `$DEV_MODE = false` - SMS будут отправляться реально

3. **Расширенное логирование:**
   - Подробные логи для отладки в файле error.log на сервере

## После загрузки:

1. Проверьте работу регистрации (должен приходить SMS с кодом)
2. Проверьте оформление заказа (должен приходить SMS с подтверждением заказа)

## Если SMS не приходят:

1. Проверьте логи на сервере (обычно в `/var/log/apache2/error.log` или `/var/log/nginx/error.log`)
2. Убедитесь, что учетные данные OsonSMS правильные:
   - Login: cafe Simin
   - Hash: 17e6c7a8ee5ae228610e0c62554aefa9
   - Sender: Cafe Simin

3. Свяжитесь с поддержкой OsonSMS для проверки баланса и статуса аккаунта
